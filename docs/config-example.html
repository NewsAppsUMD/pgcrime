Data Path Configuration Examples

Option 1: Inline Configuration (Before loading app.js)

<script>
    // Set custom data path before loading app.js
    window.DATA_PATH = '/custom/path/to/data/json/';
</script>
<script src="app.js"></script>

Option 2: Common Configurations

Default (Relative Path)

window.DATA_PATH = '../data/json/';

Use when website is in website/ and data is in data/json/

Same Directory

window.DATA_PATH = './data/json/';

Use when data folder is in the same directory as index.html

Absolute Path (Local)

window.DATA_PATH = '/data/json/';

Use for absolute paths from web root

Remote API or CDN

window.DATA_PATH = 'https://example.com/api/crime-data/';

Use if serving data from a different domain (requires CORS)

Testing Your Configuration

After setting the path, test it by:

  1. Open test.html to validate data accessibility
  2. Check browser console for any fetch errors
  3. Verify the correct path is displayed in test.html

Current Setup Detection

The dashboard will automatically log the data path being used. Check your browser console for messages like:

Dashboard initialized with data path: ../data/json/
